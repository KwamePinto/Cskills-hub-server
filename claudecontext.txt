--- KEY DESIGN DECISIONS TO REMEMBER ---

1. DENORMALIZED COUNTS on Course (totalLessons, totalQuizzes, enrollmentCount)
   WHY: Avoids COUNT queries on every course list request.
   HOW: Increment/decrement these fields when lessons/quizzes are added/removed.

2. DIRECT COURSE REFERENCE on Lesson and Quiz (lesson.course = courseId)
   WHY: Allows querying all lessons for a course without joining Chapter first.
   PERFORMANCE: Avoids N+1: course в†’ chapters в†’ lessons chain.

3. BLOCK SYSTEM in Chapter schema (blocks: [{blockType, refId, order}])
   WHY: Enables modular course building вЂ” admins insert lessons/quizzes in any order.
   HOW: blockType tells us whether refId points to Lesson or Quiz collection.

4. CACHED completionPercentage in Progress
   WHY: Avoids recomputing percentage on every progress read.
   HOW: Recalculate only when a lesson/quiz is completed, not on every GET.

5. correctAnswer NEVER sent to client in Quiz GET
   WHY: Security вЂ” prevents answer scraping.
   HOW: Controller omits correctAnswer and isCorrect from question options in response.

6. COMPOUND UNIQUE INDEX on Enrollment [user, course]
   WHY: Database-level guarantee that a user cannot enroll in same course twice.

7. SEPARATE Admin model (not just a role on User)
   WHY: Admins are a fundamentally different entity. Keeps concerns clean.
   BENEFIT: No risk of user accidentally gaining admin access via role field.

8. SETTINGS as single document (upserted)
   WHY: Platform settings are one global record, not a collection of rows.
   HOW: Use findOneAndUpdate with upsert:true so there's always exactly one doc.

--- PACKAGE REFERENCE NOTES ---

express-validator:
  - body('field').notEmpty().withMessage('...')  вЂ” validate body fields
  - validationResult(req) вЂ” collect validation errors
  - Always run validation before controller logic

bcryptjs:
  - bcrypt.hash(password, 10) вЂ” hash with salt rounds = 10
  - bcrypt.compare(plaintext, hashed) вЂ” verify on login
  - Never store plain passwords

jsonwebtoken (JWT):
  - jwt.sign(payload, secret, { expiresIn }) вЂ” create token
  - jwt.verify(token, secret) вЂ” decode and validate
  - Access token: short-lived (15m), Refresh token: long-lived (7d)

Mongoose:
  - select: false on password field вЂ” never return it by default
  - pre('save') hooks for hashing passwords
  - Indexes defined in schema for performance
  - lean() on read-only queries for raw JS objects (faster than full Mongoose docs)

Multer:
  - memoryStorage() вЂ” keep file in memory buffer before sending to Cloudinary
  - limits: { fileSize } вЂ” prevent huge uploads
  - fileFilter вЂ” whitelist allowed MIME types
